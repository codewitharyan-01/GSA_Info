Q.1 [14 Marks]

(a) Write short note on Indian IT ACT 2000. [3 Marks]

Indian IT ACT 2000

What it is: The Information Technology Act, 2000 is India's main law for dealing with cyber crimes and electronic commerce. It's like a rulebook for using computers and the internet in India.

Key Features:
  • Legal recognition of electronic documents and digital signatures
  • Defines cyber crimes and their punishments
  • Establishment of CERT-In (Computer Emergency Response Team)
  • Regulates certifying authorities for digital certificates
Important Sections:
  • Section 43: Penalty for damage to computer systems
  • Section 66: Computer related offenses
  • Section 67: Publishing obscene information
  • Section 72: Breach of confidentiality
Why it matters:

It provides legal framework for safe and secure digital transactions, protects against cyber crimes, and promotes e-commerce in India.

(b) What is the difference between Threat, Vulnerability, and Risk and Computer Virus? [4 Marks]

Threat, Vulnerability & Risk

Threat

A potential danger that could exploit a vulnerability. Like a thief trying to break into your house.

Examples: Hacker, malware, natural disaster
Vulnerability

A weakness in your system that a threat could exploit. Like having a broken window in your house.

Examples: Weak password, unpatched software, open port
Risk

The chance that a threat will successfully exploit a vulnerability and cause damage. Like the probability of the thief actually breaking in.

Formula: Risk = Threat × Vulnerability × Impact

Computer Virus

What is a Virus?

A computer virus is like a biological virus - it's a malicious program that attaches itself to other programs and spreads from one computer to another, causing damage.

How it works:
  • Attaches to executable files
  • Activates when host program runs
  • Replicates and spreads to other files
  • Can damage files, steal data, or crash systems
Types:
  • File Virus: Infects executable files
  • Boot Sector Virus: Infects startup files
  • Macro Virus: Infects documents
  • Polymorphic Virus: Changes its code

(c) What is the need of Demilitarized Zone (DMZ)? Explain with example. [7 Marks]

Demilitarized Zone (DMZ)

A DMZ is like a neutral area between two countries at war. In networking, it's a special network zone that sits between your internal network (trusted) and the internet (untrusted), providing an extra layer of security.

Why DMZ is Needed

1. Extra Security Layer

Creates buffer zone between internet and internal network

2. Public Services

Allows external access to public services while protecting internal network

3. Controlled Access

Limits what external users can see and access

4. Attack Isolation

If DMZ is compromised, internal network remains protected

Real-World Example

Company XYZ Setup:
Internet: Untrusted external network
Firewall 1: Filters incoming traffic
DMZ: Web server, Email server
Firewall 2: Protects internal network
Internal Network: Employee computers, databases
How it works:
  • External users access web server in DMZ
  • DMZ servers can communicate with internal database if needed
  • Internal users access internet through both firewalls
  • If web server is hacked, internal network is still protected

DMZ Benefits:

Security: Multiple layers of protection
Flexibility: Public services accessible
Control: Monitored and filtered access

Q.2 [14 Marks]

(a) Define the term in briefly: [3 Marks]
(i) Open Port Identification (ii) Banner Check

(i) Open Port Identification

What it is: Checking which doors (ports) are open on a computer system to see what services are running.

Why it's important:
  • Finds security weaknesses
  • Identifies running services
  • Helps in security assessment
  • Detects unauthorized services
Example:

Port 80 (HTTP) open = Web server running
Port 22 (SSH) open = Remote access available

(ii) Banner Check

What it is: Reading the welcome message (banner) that services show when you connect to them. It's like reading a nameplate on an office door.

What banners reveal:
  • Software name and version
  • Operating system details
  • Service configuration
  • Server information
Security Risk:

Banners can give attackers information about vulnerabilities in specific software versions, helping them plan attacks.

(b) What is Probe? Explain its different types. [4 Marks]

What is a Probe?

A probe is like a scout or detective that gathers information about a computer system or network. It's a tool or technique used to explore and collect data about target systems.

Active Probes

Definition: Directly interact with the target system to get information.

Types:
  • Ping Probe: Checks if system is alive
  • Port Scan: Finds open ports and services
  • Version Detection: Identifies software versions
  • OS Detection: Determines operating system
Characteristics:
  • More accurate information
  • Can be detected by target
  • May trigger security alerts
  • Direct interaction required

Passive Probes

Definition: Observe network traffic without directly interacting with the target.

Types:
  • Traffic Analysis: Studies communication patterns
  • Packet Sniffing: Captures network packets
  • DNS Monitoring: Watches DNS queries
  • Social Engineering: Gathers information from people
Characteristics:
  • Harder to detect
  • Less accurate information
  • Doesn't trigger alerts
  • Indirect information gathering

(c) List and explain types of Network Sniffer. List Network Sniffers and Injection tools. [7 Marks]

Network Sniffer

A network sniffer is like a wiretap for computer networks. It captures and analyzes data traveling through a network, helping administrators troubleshoot problems or attackers steal information.

Types of Network Sniffers

1. Passive Sniffers

Only capture traffic without interfering. Like a security camera that just records.

Use: Network monitoring and troubleshooting
2. Active Sniffers

Inject packets into network to gather more information. Like asking questions to get answers.

Use: Security testing and network discovery
3. Filter-Based Sniffers

Capture only specific types of traffic based on rules. Like a filter that only catches fish of certain size.

Use: Targeted monitoring and analysis
4. Protocol Analyzers

Deep inspection of specific protocols. Like a translator for network languages.

Use: Protocol debugging and security analysis

Q.5 [14 Marks]

(a) What is Brute-Force Attack? How it can be prevented? [3 Marks]

Brute-Force Attack

What it is: A brute-force attack is like trying every possible key to open a lock. The attacker systematically tries all possible combinations of passwords or encryption keys until they find the correct one.

How it works:
  • Automated tools try thousands of passwords per second
  • Starts with common passwords (123456, password)
  • Tries dictionary words, then random combinations
  • Continues until correct password is found
Prevention Methods:
Strong Passwords: Use long, complex passwords with mixed characters
Account Lockout: Lock account after failed attempts
CAPTCHA: Add human verification challenges
Two-Factor Auth: Require second verification method
Rate Limiting: Slow down repeated attempts
Password Managers: Generate and store complex passwords

(c) Explain how SQL Injection attacks can be prevented. [7 Marks]

SQL Injection Prevention

SQL injection is like someone sneaking commands into a conversation to trick the database. Prevention methods are like security checks to ensure only valid requests reach the database.

1. Input Validation

Check all user input before processing. Like security screening at airports.

Methods: Whitelist allowed characters, validate data types, check length limits

2. Parameterized Queries

Use placeholders instead of directly inserting user input into SQL queries.

Example: "SELECT * FROM users WHERE id = ?" instead of "SELECT * FROM users WHERE id = " + userInput

3. Stored Procedures

Use pre-defined database procedures that don't accept raw SQL.

Benefit: Separates application logic from database queries

4. Least Privilege

Give database users only the minimum permissions they need.

Example: Read-only access for display pages, no delete permissions for normal users

5. Web Application Firewall (WAF)

Firewall that specifically protects against web-based attacks.

Function: Filters and monitors HTTP traffic between web application and internet

6. Regular Security Testing

Continuously test applications for vulnerabilities.

Methods: Penetration testing, code reviews, automated security scanning

Best Practices Summary:

Never trust user input
Always validate and sanitize
Use prepared statements
Separate code from data
Implement multiple layers
Defense in depth approach
